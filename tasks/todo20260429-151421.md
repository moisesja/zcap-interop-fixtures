## Review

### What changed

- `python/interop_fixtures/adapters.py` — `_reference_capability_payload`
  and `_reference_invocation_payload` rewritten to the flat W3C Data
  Integrity shape: document fields at the top level + `proof` minus
  `proofValue`. The hand-picked proof-field whitelist is gone, so all
  proof fields (e.g. `nonce`, `domain`) participate in JCS.
- Four new isolation fixtures committed (two for the wrapper-vs-flat
  divergence, two for the proof-field-whitelist divergence).
- `deficiencies.md` gains a `Cross-Library JCS Payload Shape` top-level
  section.
- `docs/periodic-runs.md` gains a `Release-Blocking Fixtures` section
  pointing at the four new fixtures.
- `CHANGELOG.md` updated under `## Unreleased`.

### Verification

- `python3 -m compileall python` is clean.
- Reference manifest builds against all 23 fixtures (19 existing + 4
  new); every entry has `status == "ok"`.
- Confirmed flat shape on existing fixtures: `capability-minimal`'s
  canonical bytes now begin `{"@context":...` rather than
  `{"capability":{"@context":...`. Same for `invocation-minimal`.
- Diffed wrapped vs flat SHA-256 for `capability-minimal`:
  - wrapped (old): `bf1065253d5d9243537e21b316567a9c17ca2808ddc526c5a11269d8ab687ebc`
  - flat (new):    `0d74b28bba150b2834604f90c205fa7a42f3ca6c4af671bbdba0d30f969af859`
  Bytes actually changed — the flip is real.
- Each new fixture key set matches `schema/interop-fixture.schema.json`
  exactly (no missing required fields, no extra fields).

### Expected matrix behavior after this change

- `reference-jcs` ↔ `python-app` (`identity-authorization-service`):
  expected to **converge** (both flat once both consume the W3C shape).
- `reference-jcs` ↔ `dotnet-zcap`: expected to **diverge** on every
  fixture (wrapped envelope vs flat). This is the intended state until
  `moisesja/zcap-dotnet#34` lands. The `wrapper-vs-flat-isolation`
  fixture pair surfaces the divergence even without other fixtures.
- `proof-field-whitelist` fixture pair additionally surfaces the
  secondary divergence whenever an adapter strips proof fields.

- Programmatic CI release gating — only documentation of the
